# Datasets in OneEHR (Decoupled Design)

OneEHR **does not** manage a registry of dataset names (e.g. TJH/MIMIC/etc).
Instead, OneEHR consumes a single, unified **event table** format. Any external
dataset can be used as long as you can export/convert it into this format.

## Unified Event Table (Required)

Minimum required columns (configurable names via `[dataset]`):

- `patient_id` (`dataset.patient_id_col`)
- `event_time` (`dataset.time_col`) — must be parseable by `pandas.to_datetime`
- `code` (`dataset.code_col`) — concept / measurement name
- `value` (`dataset.value_col`) — numeric or categorical value

Anything else can remain as extra columns; they can be useful for:

- static features (`[static_features]`)
- label generation (`[labels] fn = "path/to/label_fn.py:build_labels"`)

## Two Ways to Use a Dataset

### A) Recommended: Convert outside OneEHR (fully decoupled)

Write a script that reads raw files and writes a normalized `events.csv`:

- `scripts/convert_my_dataset.py` (your project code, not OneEHR)
- output: `events.csv` in unified schema

Then your TOML only needs:

```toml
[dataset]
path = "/path/to/events.csv"
file_type = "csv"
```

### B) Optional: Use `converter_fn` (glue code hook)

If you don’t want to materialize `events.csv` on disk, you can point OneEHR at a
raw file and provide a converter callable:

```toml
[dataset]
path = "/path/to/raw.xlsx"
converter_fn = "path/to/converter.py:convert"
```

Converter signature:

```python
def convert(df_raw: pd.DataFrame, cfg: DatasetConfig) -> pd.DataFrame | ConvertedDataset:
    ...
```

Return value must contain the unified event table as a `pd.DataFrame` (either
directly, or wrapped via `ConvertedDataset(events=...)`).

## Labels

OneEHR does **not** require labels to be “registered” per dataset. Labels are
generated by a `label_fn` that reads columns from the event table:

```toml
[labels]
fn = "path/to/label_fn.py:build_labels"
```

- N-1 (patient mode): return columns `patient_id`, `label`
- N-N (time mode): return `patient_id`, plus `bin_time` or a time column to bin
