# OneEHR Architecture

OneEHR is organized around an end-to-end EHR modeling workflow:

1. **Data pre-processing**: load raw event tables (CSV/Excel), parse timestamps, type values, and
   discretize into fixed bins (user-configurable `bin_size`).
2. **Modeling**: train ML or DL models on the processed representations.
3. **Evaluation**: compute task-appropriate metrics and produce paper-style result tables.
4. **Analysis** (future): interpretability, uncertainty, confidence, etc.

Key design goals:

- **User usability first**: doctor-friendly inputs; minimal required columns.
- **No leakage**: patient-level grouping is enforced for all split strategies.
- **Extensibility**: labels can be generated by user-provided `label_fn` in TOML `[labels]`.
