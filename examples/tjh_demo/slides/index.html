KDD Hands-on Tutorial

OneEHR

Real TJH data, end-to-end modeling, held-out testing, agent workflows, case bundles, and a polished web console in one demo.

01

What This Demo Shows

  • Real dataset: TJH train cohort plus external held-out cohort.
  • Full chain: preprocess → train → test → analyze → agent predict → cases → agent review.
  • Production-style UX: one web UI for model results, test audit, comparison, and agent outputs.
  • LLM integration: Zenmux OpenAI-compatible backend with openai/gpt-5.4.

Tutorial Theme

From raw longitudinal EHR tables to an interpretable, reviewable, demo-ready operations console.
OneEHR turns offline experiments into artifacts that are easy to inspect live.

System View

OneEHR Tutorial Pipeline

The demo is structured so every stage leaves durable artifacts that feed the next stage and stay explorable in the UI.

02
Stage 1

Preprocess

Standardize TJH raw Excel into train and external-test CSV tables.

Stage 2

Train

Fit tabular baselines on the train cohort with durable manifests and split outputs.

Stage 3

Test

Run held-out evaluation on the external cohort and persist test summaries.

Stage 4

Analyze

Emit dataset, cohort, prediction, test, temporal, interpretability, and agent audit modules.

Stage 5

Predict

Call Zenmux-backed LLM predictors over saved patient instances.

Stage 6

Cases

Materialize case bundles with timeline, static context, predictions, and analysis references.

Stage 7

Review

Run agent review and compare model, external-test, and agent deltas in one console.

Every stage stays queryable from the web UI and the CLI.
